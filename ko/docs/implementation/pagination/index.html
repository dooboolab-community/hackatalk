페이지네이션

Pagination은 graphql을 사용할 때 도전적인 과제일 수 있습니다. 페이스북에서 제공한 relay cursor pagination을 활용하면 현재 많은 모바일 화면에서 제공되는 무한 스크롤 리스트뷰를 구현하는데 매우 용이합니다.

하지만 relay cursor pagination 또한 위에서 보이는 offset pagination을 구현하는데 한계점들이 존재합니다. 하지만 현재(2021년 6월 25일)는 모바일에서 제공되는 무한 스크롤뷰를 구현하기 위해 cursor pagination이 가장 이상적인 방법입니다.

서버#

과거에는 relay cursor pagination을 sequelize를 이용하여 구현하면 복잡한 상황을 초래했습니다. 관련하여 과거 경험을 미디엄 포스트에서 읽어볼 수 있습니다.

하지만, 우리는 이제 prisma2와 nexus로 마이그레이션을 완료하였고 이제는 더욱 편리하게 relay cursor paginiation을 적용할 수 있습니다. 심지어 nexus는 connection plugin을 제공하며 해커톡에서 이를 사용하고 있습니다.

클라이언트#

클라이언트에서는 relay를 사용하여 서버의 graphql api를 소비하고 있습니다. 해당 글에서는 리액트에서 relay cursor pagination을 사용하는 방법에 대해 설명하고 있습니다.
